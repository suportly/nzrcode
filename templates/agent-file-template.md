## Before you write code

Invoke the right skill for the stage you are at:

| Stage | Skill |
|---|---|
| User described a demand, no `spec.md` yet | `specify` |
| `spec.md` exists, has `[NEEDS CLARIFICATION]` markers | `clarify` |
| Spec is clean, no `plan.md` yet | `plan` |
| `plan.md` is approved, no `tasks.md` yet | `tasks` |
| `tasks.md` is ready | `implement` |
| Something looks off between spec, plan, and code | `analyze` |
| A specific category (security / perf / a11y) needs a pass | `checklist` |
| Test failing or unexpected behavior | `systematic-debugging` |
| Ready to open a PR | `requesting-code-review` |
| Review approved | `finishing-a-branch` |

`brainstorming` and `writing-plans` were removed in 0.2. Use `specify` +
`plan` + `tasks` instead — one artifact per stage, validated against the
constitution.

## Things this agent file deliberately does **not** do

- It does not tell you to check every skill at a 1% threshold. Use judgment:
  the skill rule gates **write actions**, not questions or exploration.
- It does not repeat the constitution verbatim. It links to it.
- It does not hard-code the stack. Stack-specific guidance lives in the
  active preset's files so switching presets updates this file automatically.
